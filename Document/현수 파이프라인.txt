1. 데이터 모음 - 크롤링을 통해 모은 데이터와 회사에서 보유한 데이터를 합쳐서 데이터 셋을 구축한다. 혹시 데이터가 부족하다면 2020 대한 영상의학회지에 실린 논문을 바탕으로 가장 높은 성능을 보인 랜덤회전 기법 이나 기존 데이터를 기반으로 완전히 새로운 데이터를 만들어 내는 GAN을 사용하면 좋을 것이라고 생각한다.

2. 데이터 전처리 - 우선 mobilenet 모델을 사용할 것이기 때문에 mobilenet모델의 처음 들어가는 이미지의 사이즈인 224*224로 조정해준다. 이후 각 픽셀값을 0~1사이 숫자로 변환해 normalization 해준다(값이 큰 feature의 영향을 줄이기 위해[이를 통해 local min or max에서 멈추는 것을 방지]) 이미지 필터의 경우 크롤링한 이미지를 필터적용한 것을 보았을때 Sharpening기법을 사용했을때 치아를 선명하게 볼 수 있었기에 해당 기법을 사용했으면 좋겠다.

3. 데이터셋 분할 - 현재 코드에서는 랜덤으로 가지고 온 데이터를 학습용과 검증용으로 9:1 비율로 분리한 후 다시 학습용 데이터를 학습용과 테스트용으로 분리하는데 순서를 바꿔 학습용과 테스트용으로 9:1비율로 분리하고 다시 학습용 데이터를 학습요과 검증용으로 8:2로 분리하여 대세인 8:2 분리를 따르며 테스트용 테이터를 전체의 10%로 잡아 학습용이든 테스트용이든 데이터가 부족하지 않게 해주면 좋을 것 같다.

4. 모델학습 - 우선 모델의 경우 mobilenet을 사용하고 가중치는 imagenet으로 이전에 학습한 값을 사용한다. 이미지 데이터 즉 2차원 데이터를 사용하기 때문에 flatten layer를 사용하게 되고 이에 따라 파라미터 수가 늘어남으로 코드처음 global average pooling을 활용한다. 과적합 방지를 위해 drop out을 사용하는데 이 경우 리딩 시간에 발표했던 논문을 보면 비슷한 모델로 0.2 또는 0.25 정도로 좋은 결과를 나타냈기에 성능이 좋지 않으면 저렇게 바꿔봤으면 좋겠고 반대로 성능이 좋으면 보편적으로 사용하는 0.5로 조절하여 모델의 무게를 줄였으면 좋겠다. 또한 우리는 정상, 치주염 초기, 중기, 말기 4가지 class로 분류하기 때문에 multi-class classification에 사용할 수 있는 활성화 함수 softmax를 활용한다.(0~1 사이로 정규화해서 값을 주는 이점도 있다)/*코드또한 Dense(4, activation='softmax'로 바꿔야 할듯)*/ 
모델의 loss함수의 경우 다중 분류이며 각 픽셀 데이터를 0~1사이로 normaliztion했기 때문에 Categorical Crossentropy를 활용하고 모델 최적화의 경우 가장 보편적으로 많이 사용하는 adam을 사용하고 ADAM: A METHOD FOR STOCHASTIC OPTIMIZATION(2015)에 따라 학습률은 0.001, decay는 경량화 모델이다보니 처음에는 0.0으로 설정하고 과적합 문제가 생긴다면 이후 수정해도 괜찮을 것 같다.

5. 평가 - 현재 코드에는 평가를 f1-score 만으로 하는데 아무래도 의학적인 의사결정이기 때문에 실제 정답 중 얼마나 맞췄는지 보여주는 recall 또한 (f1-score에 recall 역시 일부 포함되지만) 평가지표로 사용했으면 좋겠다. 또한 학습 검증 데이터의 구간별 정확도와 손실을 표로 나타내어 하이퍼파라미터 수정에 도움이 될 수 있도록 해봤으면 좋겠다.
